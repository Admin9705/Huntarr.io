Docker Installation

The recommended way to install Huntarr.io is using Docker. This method simplifies the installation process and ensures that all dependencies are properly managed.

Using Docker Compose (Recommended)

Create a docker-compose.yml file with the following content:

version: '3'
services:
  huntarr:
    container_name: huntarr
    image: plexguide/huntarr:latest
    restart: unless-stopped
    ports:
      - "9705:9705"
    volumes:
      - ./config:/config
      - /etc/localtime:/etc/localtime:ro
    environment:
      - PUID=1000
      - PGID=1000
      - TZ=Europe/London  # Set to your timezone

Then run the following commands:

mkdir -p config
docker-compose up -d

Using Docker Run

If you prefer using the docker run command:

docker run -d \
  --name=huntarr \
  -p 9705:9705 \
  -v $(pwd)/config:/config \
  -v /etc/localtime:/etc/localtime:ro \
  -e PUID=1000 \
  -e PGID=1000 \
  -e TZ=Europe/London \
  --restart unless-stopped \
  plexguide/huntarr:latest

Unraid Installation

Huntarr.io can be easily installed on Unraid using the Community Applications plugin.

Steps for Unraid Installation

  1. Navigate to the Apps tab in your Unraid dashboard
  2. Search for "Huntarr" in the search bar
  3. Click on the Huntarr app
  4. Configure the container settings:
    • Set your preferred port (default: 9705)
    • Set the config directory path
    • Set your PUID, PGID, and timezone
  5. Click Apply to create the container

Native Installation

For advanced users who prefer to install Huntarr.io directly on their system:

  1. Ensure you have Python 3.9+ and Git installed on your system
  2. Clone the repository: 
    git clone https://github.com/plexguide/Huntarr.io.git
  3. Navigate to the directory: 
    cd Huntarr.io
  4. Install Python dependencies: 
    pip install -r requirements.txt
  5. Run the application: 
    python run.py

macOS Installation

Important macOS Permission Warning

The Huntarr macOS application has known permission issues when attempting to write to the ~/Library/Application Support/ directory. This can cause crashes and data loss.

Recommended Configuration for macOS

To avoid permission issues on macOS, we've implemented a fallback system that:

  1. First tries to use the app bundle's own Resources directory for configuration files
  2. Falls back to the user's Documents folder if that fails

For the best experience on macOS, use the following installation steps:

  1. Download the latest macOS installer from the GitHub releases page
  2. Move the application to your Applications folder
  3. When running for the first time, right-click and select "Open" to bypass Gatekeeper
  4. The application will automatically create its configuration in the appropriate location

Troubleshooting macOS Issues

If you encounter crashes or permission errors, check the application logs located in:

~/Library/Logs/Huntarr/ or ~/Documents/Huntarr/logs/

Post Installation

Initial Setup

After installing Huntarr.io, you'll need to complete the initial setup:

  1. Access the web interface (default: http://localhost:9705)
  2. Create an administrator account when prompted
  3. Follow the setup wizard to configure your media paths and integrate with media applications (Sonarr, Radarr, etc.)

Troubleshooting

Common Issues

Port already in use

If port 9705 is already in use, you can change it in the docker-compose.yml file or use the --port option in the installation script.

Connectivity problems

If you can't connect to Huntarr.io after installation:

  • Check if the service is running: docker ps | grep huntarr
  • Check the logs: docker logs huntarr
  • Ensure your firewall allows traffic on the configured port
